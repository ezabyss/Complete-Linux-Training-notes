# 🧠 PS Command in Linux (Process Status)

`ps` stands for **Process Status**.

It displays information about currently running processes in your Linux system.

---

# 📌 What Happens When You Run a Command?

Every time you:

- Start an application
- Run a command
- Open a shell

Linux creates a **process** with a unique **PID (Process ID)**.

The `ps` command allows you to view those processes.

---

# 🔹 Basic PS Command

Run:

`ps`

This shows processes related to your **current shell session** only.

---

## 🧾 Default Output Columns

When you run `ps`, you will see something like:

- PID
- TTY
- TIME
- CMD

### Column Explanation:

| Column | Meaning |
|---------|----------|
| PID | Process ID (unique number) |
| TTY | Terminal associated with process |
| TIME | CPU time used |
| CMD | Command name |

If you see `00:00:00` under TIME, it means the process hasn’t used noticeable CPU time yet.

---

# 🔹 Show All Processes

To see all running processes:

`ps -e`

This lists every running process in the system.

---

# 🔹 BSD Style Format (Detailed View)

`ps aux`

This gives a much more detailed output.

Columns include:

- USER
- PID
- %CPU
- %MEM
- VSZ
- RSS
- START
- TIME
- COMMAND

This format is very commonly used by system administrators.

---

# 🔹 Full Format Listing (Very Popular)

`ps -ef`

This is one of the most widely used formats.

It shows:

- UID (user)
- PID
- PPID (parent process ID)
- Start time
- TTY
- CMD

Many Linux engineers prefer `ps -ef` for troubleshooting.

---

# 🔹 Show Processes by Specific User

To see processes owned by root:

`ps -u root`

To see processes owned by your user:

`ps -u username`

Example:

`ps -u ezabyss`

This is useful for monitoring user activity.

---

# 🔹 Search for a Specific Process

Instead of scanning everything, you can filter:

`ps -ef | grep firewalld`

Or partial match:

`ps -ef | grep fire`

This helps when you don’t remember the full process name.

---

# 🔹 Compare with Windows

In Windows:

- You open **Task Manager**
- View running applications and services

In Linux:

- `ps`
- `top`
- `systemctl status service_name`

These commands provide similar monitoring capabilities.

---

# 🔹 Example: Checking a Service Process

Check service status:

`systemctl status firewalld`

Note the PID.

Then verify using:

`ps -ef | grep 750`

Or:

`ps -ef | grep firewalld`

Both methods confirm the running process.

---

# 🧠 Summary of Important PS Options

| Command | Purpose |
|----------|----------|
| `ps` | Show current shell processes |
| `ps -e` | Show all processes |
| `ps aux` | Detailed BSD format |
| `ps -ef` | Full-format listing |
| `ps -u user` | Show processes by user |
| `ps -ef \| grep name` | Search specific process |

---

# 🎯 Key Takeaway

- Every running program becomes a process.
- `ps` lets you inspect those processes.
- `ps -ef` is commonly used in production environments.
- Combine with `grep` to filter results quickly.

Understanding `ps` is foundational for:

- Troubleshooting
- Monitoring
- Killing processes
- Performance analysis
